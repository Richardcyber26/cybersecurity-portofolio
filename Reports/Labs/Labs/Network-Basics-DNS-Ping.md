Network Basics - DNS & Ping Analysis

Objective

To understand how DNS resolution and network connectivity work using basic command-line tools.

Tools Used

- Command Prompt (Windows 11)
- nslookup
- ping

Observations

1. DNS Resolution

The command "nslookup google.com" returned multiple IPv4 and IPv6 addresses. This shows that large websites use multiple servers for load balancing and redundancy.

2. Network Connectivity

The "ping google.com" command showed successful responses with 0% packet loss. The average response time was around 142ms, indicating stable internet connectivity.

3. DNS Server

The DNS server used was shown as "UnKnown", which is normal for ISP or router-based DNS systems.

Conclusion

This exercise demonstrates how domain names are translated into IP addresses and how network connectivity can be tested using basic tools. These are fundamental skills in cybersecurity and network analysis.
